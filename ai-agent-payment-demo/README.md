## 安全特性

### AI会话密钥系统
- **随机生成**: 每次会话使用随机ID和令牌
- **权限控制**: AI只能执行授权的支付操作
- **有效期**: 会话有24小时有效期
- **防重放**: 每个会话只能使用一次

### 链上支付验证
- **真实交易**: 所有支付都是真实的ETH转账
- **区块链验证**: 交易哈希可在Etherscan验证
- **余额检查**: 支付前检查ETH余额
- **Gas估算**: 自动估算合理的Gas费用

## 验证真实支付

支付完成后，系统会提供：
1. **交易哈希** - 区块链交易唯一标识
2. **区块号** - 交易所在的区块
3. **状态** - 交易成功或失败
4. **验证链接** - Sepolia Etherscan链接

**验证步骤**:
1. 打开提供的Etherscan链接
2. 确认交易状态为"Success"
3. 验证转账金额正确
4. 确认收款地址为商户地址

## 故障排除

### 常见问题
1. **私钥错误**: 确保 `USER_PRIVATE_KEY` 环境变量正确
2. **余额不足**: 确保钱包有足够的ETH支付商品和Gas
3. **网络连接**: 检查是否能访问Sepolia RPC
4. **服务未启动**: API服务需要在端口3000-3007运行

### 测试网ETH获取
如果需要Sepolia测试网ETH：
1. 访问 [Sepolia Faucet](https://sepoliafaucet.com)
2. 输入你的钱包地址
3. 获取测试ETH

## 技术实现

### 支付流程
```javascript
// 1. 用户确认支付
用户输入"确认支付"

// 2. 生成AI会话
const session = createAISession()
// → 会话ID、令牌、权限

// 3. 验证会话
verifySession(session.id, session.token)
// → 验证通过

// 4. 执行链上支付
executeRealPayment(product, amount, session)
// → 交易哈希、区块号、状态

// 5. 访问API
accessPaidAPI(product, transactionHash)
// → 获取商品数据
```
